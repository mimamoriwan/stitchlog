# タスク: DMC全色・Olympus全色のカラーDBを自動生成する

## 背景
- 現在のDMCカラーDBは50色のみ → 24色指定しても14色に収束してしまう
- Olympusは未実装
- 両方のRGBデータがhutarigurashi.comで無料公開されている

## Step 1: Olympus Excelファイルをダウンロードして構造を確認

```bash
# pnpm add で xlsx を追加
pnpm --filter @stitchlog/conversion-engine add xlsx

# Olympus Excelをダウンロード
curl -L -o /tmp/RGB-OLYMPUS.xlsm \
  "https://hutarigurashi.com/download/4441/"

# ファイルが取得できたか確認
ls -lh /tmp/RGB-OLYMPUS.xlsm
```

次に以下のスクリプトでカラム構造を確認して報告する:

```typescript
// scripts/inspectExcel.ts として一時作成
import * as XLSX from 'xlsx';

const wb = XLSX.readFile('/tmp/RGB-OLYMPUS.xlsm');
const sheet = wb.Sheets[wb.SheetNames[0]];
const rows = XLSX.utils.sheet_to_json(sheet, { header: 1 }) as unknown[][];

// 先頭10行だけ表示
console.log('Sheet name:', wb.SheetNames[0]);
console.log('First 10 rows:');
rows.slice(0, 10).forEach((row, i) => console.log(i, row));
```

## Step 2: カラム構造の確認結果を踏まえてパーサーを作成

確認した構造に基づいて
`scripts/generateThreadDb.ts` を作成する。

出力形式は以下の既存の `ThreadColorEntry` 型に合わせること:
```typescript
// packages/conversion-engine/src/threadColors/dmc.ts の既存型
export interface ThreadColorEntry {
  brand: ThreadBrand;
  code: string;
  name: string;
  hex: string;
  r: number;
  g: number;
  b: number;
}
```

- hex は r/g/b から `#${r.toString(16).padStart(2,'0')}...` で生成する
- name は色コードをそのまま使う（日本語名があれば使う）

## Step 3: 生成スクリプトを実行

```bash
# Olympus生成
npx tsx scripts/generateThreadDb.ts olympus \
  /tmp/RGB-OLYMPUS.xlsm \
  packages/conversion-engine/src/threadColors/olympus.ts

# 生成された色数を報告（Olympusは約434色を期待）
```

## Step 4: DMC版も同様に処理

## Step 6: 動作確認

```bash
pnpm --filter @stitchlog/conversion-engine tsc --noEmit
pnpm --filter @stitchlog/web tsc --noEmit
pnpm dev
```

localhost:3001 で以下を確認してスクリーンショットを撮る:
1. 糸ブランド「Olympus」に切り替えて写真を変換
2. カラーパレットに「Olympus-XXX」形式の番号が出ること
3. 色数が指定した数（24色なら24色に近い数）になること
