# タスク: カラーパレットの重複DMCコードを修正

## 問題
k-meansで抽出した複数の色が同じDMCコードにマッチした場合、
パレットに同じ colorCode が複数出てしまいReactのkey重複警告が出る。

## 修正1: データレベルで重複を排除（colorExtractor.ts）

extractColors の return 直前に deduplicate 処理を追加する。
同じ colorCode の色はマージし、crossStitchCount を合算する。

palette.map(...) の後、return の前に以下を挿入:

// 同じDMCコードをマージ
const merged = new Map<string, typeof rawColors[0]>();
for (const c of rawColors) {
  if (merged.has(c.colorCode)) {
    const existing = merged.get(c.colorCode)!;
    existing.crossStitchCount += c.crossStitchCount;
  } else {
    merged.set(c.colorCode, { ...c });
  }
}
return Array.from(merged.values());

※ rawColors は既存の palette.map(...) の結果を入れる変数名に変更する。

## 修正2: ReactのkeyをindexベースにフォールバックSafe化（page.tsx）

257行目付近の key を以下に変更:
key={`${color.colorCode}-${index}`}

※ 修正1でデータレベルの重複は排除されるが、
  将来的な安全策としてindexを suffix に追加する。
